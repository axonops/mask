# Changelog

All notable changes to `github.com/axonops/mask` are documented in this file.

The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- Stable releases now auto-post an announcement to the GitHub Discussions Announcements category via `.github/workflows/release-announcement.yml`. The workflow triggers on `release: published`, skips pre-releases and drafts, builds the body from the release notes plus a tag-pinned footer template at `.github/DISCUSSION_ANNOUNCEMENT_FOOTER.md`, and authenticates with `GITHUB_TOKEN`. Governance test `TestGovernance_ReleaseAnnouncementWorkflow` pins the trigger, the prerelease/draft filter, the `discussions: write` + `contents: read` permissions, the Announcements category id, and the footer-template reference. ([#51](https://github.com/axonops/mask/issues/51))

### Fixed

- `date_of_birth` slash form (`D/M/YYYY`, `DD/MM/YYYY`) now width-matches the middle field to the input month, mirroring the ISO branch. The previous implementation at `rules_identity.go:404-408` hardcoded the middle field to four mask runes regardless of input width, so `15/03/1985` emitted `**/****/1985` (and `1/1/2000` emitted `*/****/2000`) — an internal inconsistency surfaced by the corpus harness in [#54](https://github.com/axonops/mask/issues/54) and originally pinned as a deliberate quirk. Reviewed and corrected on 2026-05-15: `15/03/1985` now emits `**/**/1985`, `5/3/1985` emits `*/*/1985`, and asymmetric inputs (`1/10/2000` → `*/**/2000`, `10/1/2000` → `**/*/2000`) preserve per-field width. `parseDOBSlash` returns both field widths instead of one. New unit cases in `TestApply_DateOfBirth`, new BDD scenarios in `tests/bdd/features/identity.feature`, and updated canonical pins in `tests/corpus/date_of_birth.txt`. ([#80](https://github.com/axonops/mask/issues/80))
- `connection_string` now redacts the `pass` short-form password query parameter alongside the existing `password` / `passwd` / `pwd` family. The rule's `secretQueryKeys` map at `rules_technology.go:84-117` was missing `pass`, while the corpus generator's `knownSecretKeys` list at `tests/corpus/gen/connection_string.go:88-95` always included it — the two lists had diverged and the corpus silently locked the divergence in as fail-closed pins (`postgres://host/db?pass=secret` → all asterisks) because the corpus uses the rule itself as its oracle. Surfaced during the post-#54 corpus audit. New unit tests `TestApply_ConnectionString_PassKey` (10 cases covering substring-bait safety, percent-encoded keys, multi-param positions, and the rest of the password family) and `TestApply_ConnectionString_SecretKeysSupersetGenerator` (asserts the rule's set is a superset of the generator's so this divergence cannot recur), plus a new BDD scenario and regenerated corpus fixtures. ([#82](https://github.com/axonops/mask/issues/82))
- `database_dsn` now accepts the full set of Go MySQL DSN protocol tokens — `tcp`, `tcp4`, `tcp6`, `unix`, `udp` — instead of only `tcp`. The protocol-match gate at `rules_technology.go::looksLikeDSNProtocol` was an open `[a-z]+` byte scan that stopped at the digit in `tcp4` / `tcp6` and silently rejected IPv6 deployments (`admin:secret@tcp6([2001:db8::88]:3306)/users` masked to a string of asterisks instead of redacting only the userinfo). The new matcher is a closed allowlist switch over the five recognised tokens; lowercase-only by design (`go-sql-driver/mysql` forwards the token to `net.Dial` case-sensitively). Anything outside the allowlist — `tcpx`, `quic`, `Tcp6`, `udp4`, uppercase variants — fails closed strictly, which is safer than the previous open-scan accept-anything-lowercase behaviour. Surfaced during the post-#54 corpus audit (dozens of IPv6 / unix-socket fixtures pinned fail-closed). New unit tests `TestApply_DatabaseDSN_GoMySQLProtocols` (14 cases covering every protocol plus negative pins for unknown / case-mixed / `udp4` / unterminated shapes) and `TestApply_DatabaseDSN_GoMySQLProtocols_QuerySecrets` (5 cases pinning cross-issue interaction with the #72 query-secret redaction). New BDD scenario rows and regenerated corpus fixtures. ([#83](https://github.com/axonops/mask/issues/83))
- `date_of_birth` now recognises the slash-ISO `YYYY/M/D` shape as a fourth canonical format alongside hyphen-ISO `YYYY-M-D`, slash D-first `D/M/YYYY`, and `Month D, YYYY`. The previous dispatch at `rules_identity.go::maskDateOfBirth` only recognised the D-first slash form (via `parseDOBSlash`, which rejects leading fields longer than two digits), so `2000/01/16` fell through to `SameLengthMask` and emitted ten asterisks instead of `2000/**/**`. Many real-world data feeds — Excel exports, Japanese forms, US legacy systems — write dates as `YYYY/MM/DD`. A new helper `parseDOBSlashISO` mirrors `parseDOBISO` with `/` separators; per #80 both fields width-match the input (`2000/1/6` → `2000/*/*`, `2000/01/16` → `2000/**/**`). The slash-ISO branch is dispatched before the slash D-first branch — the two grammars are syntactically disjoint so order is for clarity rather than correctness. Surfaced during the post-#54 corpus audit (50+ fail-closed fixtures). New unit test `TestApply_DateOfBirth_YYYYSlash` (25 cases covering width-matched widths, semantic-validation-free pins, dispatch-ordering pins, and a wide negative surface — mixed separators, trailing content, empty fields, non-ASCII digits, extra fields), new BDD scenario, new canonical corpus pins, and regenerated corpus. ([#84](https://github.com/axonops/mask/issues/84))
- `postal_code` now accepts UK postcodes in both spaced (`SW1A 2AA`) and spaceless (`SW1A2AA`, `BT15JE`, `EC1A7AB`) forms. The previous `maskUKPostalCode` required an ASCII space at byte `n-4`, so real-world inputs from systems that strip the separator (label printers, address-line concatenations, mobile keyboards, CSV exports) fell through to `SameLengthMask`. The fix folds the outward/inward split into a single byte-test branch: spaced inputs are unchanged; spaceless inputs are split at the fixed-width 3-byte inward suffix and the overall length floor drops from 6 to 5 to accept the shortest spaceless form (`M11AA`). Output preserves the input's spacing — `SW1A 2AA` → `SW1A ***`, `SW1A2AA` → `SW1A***`. The BS 7666 grammar gate from #71 (`isUKOutwardCode` enumerating `AN`/`ANN`/`AAN`/`AANN`/`A9A`/`AA9A`) applies to both branches, so malformed shapes still fail closed regardless of spacing. Canadian and US disambiguation: spaceless Canadian inputs like `M5V2T6` and `A1A1A1` fail the UK inward-code letter check (`d[A-Z]{2}`) and stay fail-closed; US 5-digit ZIPs hit the digits-only branch ahead of UK. Surfaced during the post-#54 corpus audit (195/300 `postal_code` fixtures previously failing closed for this reason). New unit test `TestApply_PostalCode_SpacelessUK` (28 cases pinning every BS 7666 outward shape in both spaced and spaceless forms plus the Canadian / US / lowercase / length-boundary disambiguation), new benchmark `BenchmarkApply_postal_code_uk_spaceless`, mask-char override coverage extended, new BDD scenario rows, six new canonical corpus pins, and regenerated corpus fixtures. ([#85](https://github.com/axonops/mask/issues/85))

## [1.0.1] — 2026-05-15

### Added

- OpenSSF Scorecard weekly workflow (`.github/workflows/scorecard.yml`) that publishes a supply-chain posture score to `scorecard.dev` and uploads SARIF results to the GitHub Code scanning Security tab. Triggers on push to `main`, every PR against `main`, a weekly cron (Saturday 01:30 UTC), and `workflow_dispatch` for manual verification. Governance test `TestGovernance_ScorecardWorkflowExists` asserts the workflow's permission scopes, SHA-pinned action ref, `publish_results: true`, and `paths-ignore` policy. ([#52](https://github.com/axonops/mask/issues/52))
- OpenSSF Scorecard badge in the README badge row (linking to the live viewer), and an Elsewhere link to the Scorecard report in `docs/README.md`. ([#52](https://github.com/axonops/mask/issues/52))
- Bulk fixture corpus under `tests/corpus/` with ~12,600 generated fixtures across all 68 registered rules. `connection_string` coverage spans every major database/messaging engine (PostgreSQL, MySQL/MariaDB, SQL Server, Oracle, CockroachDB, TimescaleDB, Vertica, DB2, Firebird, MongoDB, Couchbase, Cassandra, ScyllaDB, Redis, DynamoDB, etcd, Snowflake, ClickHouse, InfluxDB, Elasticsearch, Neo4j, Kafka, AMQP/RabbitMQ, Pulsar, NATS, MQTT) plus non-URL shapes (JDBC `jdbc:driver://`, libpq key-value, ADO.NET semicolon, Oracle SQLPlus and TNS) and multi-parameter query strings with secrets in every position relative to non-secret structural params. The harness sits behind the `corpus` build tag and runs as part of `make check` and the new `Corpus fixture tests` CI job. A deterministic per-rule generator framework (`tests/corpus/gen/`, build tag `corpusgen`) preserves a hand-written canonical section in each fixture file and rewrites a much larger generated section. A `.corpus.lock` SHA-256 manifest is verified at `TestMain`, and the `Corpus lock is fresh` CI job re-runs `make corpus-regen` and fails on diff so generator output stays byte-stable. Two author helpers (`tests/corpus/_tools/bootstrap` and `tests/corpus/_tools/expect`, build tag `corpushelper`) streamline canonical-section curation. ([#54](https://github.com/axonops/mask/issues/54))

### Fixed

- `phone_number` and `mobile_phone_number` now recognise the ITU-T `00<CC>` international access prefix in addition to `+<CC>`. Inputs like `0044 7911 123456` mask to `0044 **** **3456` (country code preserved, subscriber masked) instead of failing closed. The `00` prefix is kept verbatim — it is not rewritten to `+`. Inputs with a single domestic leading `0` (e.g. `07911 123456`) are unaffected. ([#55](https://github.com/axonops/mask/issues/55))
- Compact form (`00CC<digits>` with no separator between country code and subscriber) is accepted on the `00` path to match the dial-string convention. The `+` parser continues to require a separator after the country code; this asymmetry is deliberate and documented in the rule godoc. ([#55](https://github.com/axonops/mask/issues/55))
- `mask.Apply` now calls `ensureInit` on every invocation rather than only when the rules pointer is nil. `ensureInit` uses two `sync.Once` calls (`initOnce` stores an empty rule map; `builtinsOnce` registers the built-ins) and the previous fast-path skipped both whenever the rules pointer had already been published — so a parallel first-Apply caller that observed the pointer between the two `Once` calls saw an empty registry and fell through to `FullRedactMarker` for every rule. The symmetric `loadRules` path (`HasRule`, `Rules`, `Describe`, `DescribeAll`) had the same race and now also calls `ensureInit` unconditionally. `sync.Once`'s post-init fast path is one atomic load + branch, so the cost of removing the gap is negligible. Surfaced by the new corpus harness, whose parallel subtests are the first thing in the suite to exercise zero-value `Masker` init under concurrency. Regression tests: `TestZeroValueMasker_ParallelFirstApply`, `TestZeroValueMasker_ParallelFirstHasRule`. ([#54](https://github.com/axonops/mask/issues/54))
- `postal_code` UK outward-code validator no longer accepts grammatical malformations like `A1AA`, `A11A`, or `AAAA`. The previous implementation at `rules_telecom.go:391` required only "first byte letter, rest `[A-Z|0-9]` with at least one digit", which over-accepts. The new `isUKOutwardCode` enumerates the six BS 7666 patterns explicitly: `AN`, `ANN`, `AAN`, `AANN`, `A9A`, `AA9A`. Existing valid shapes (`M1`, `B33`, `CR2`, `DN55`, `W1A`, `SW1A`, `EC1A`) continue to redact as before. Surfaced by the corpus harness in [#54](https://github.com/axonops/mask/issues/54); fixed inline rather than deferred per the project's fix-don't-defer policy. New unit test `TestPostalCode_UKOutwardCodeValidation` pins each pattern. ([#71](https://github.com/axonops/mask/issues/71))

### Security

- Release artefacts now ship with SLSA/Sigstore build-provenance attestations produced by `actions/attest-build-provenance` in `.github/workflows/release.yml`. The goreleaser job declares the `id-token: write` and `attestations: write` permissions it needs to mint an OIDC token and upload an attestation; the step is gated on `inputs.dry_run != true` so validation dispatches never invoke Sigstore, and wrapped in `continue-on-error: true` so a transient Sigstore/Fulcio outage cannot block a tag from going out. Consumers verify with `gh attestation verify <artefact> --owner axonops`; `SECURITY.md` gains a "Verifying a release" section documenting the command, the raw-attestation inspection path, and the dependency on `gh` 2.49+. Governance test `TestGovernance_ReleaseAttestations` pins the workflow contract. ([#53](https://github.com/axonops/mask/issues/53))
- `database_dsn` now redacts the values of secret query parameters in the `?key=value&...` tail in addition to userinfo. The previous implementation at `rules_technology.go:1038` rewrote `user:password@` to `****:****@` but appended the query string verbatim, so a DSN like `user:secret@tcp(host:3306)/db?charset=utf8mb4&password=other&parseTime=true` leaked the second password. The fix reuses the curated `secretQueryKeys` map and `isSecretKey` helper from `connection_string` ([#70](https://github.com/axonops/mask/issues/70)) so the keyword list stays in one place. Surfaced by the multi-param corpus pins in [#54](https://github.com/axonops/mask/issues/54); fixed in [#72](https://github.com/axonops/mask/issues/72). New unit test `TestApply_DatabaseDSN_QueryParamSecrets` with 16 cases covers every secret-position permutation, OAuth/AWS keys, upper-case keys, percent-encoded-key decoding (bypass-resistance pin), substring-bait safety, empty values, unix-socket DSNs, and the bare-flag length-mask fail-closed contract. ([#72](https://github.com/axonops/mask/issues/72))
- `connection_string` now redacts the values of OAuth (`client_secret`, `client_credentials`, `refresh_token`, `id_token`), AWS (`aws_secret_access_key`), Azure (`connectionstring`, `sas`, `sastoken`), cryptographic signature (`signature`, `sig`), generic credential (`private_key`, `authorization`, `bearer`) query parameters in addition to the previously-curated password family. The under-specified secret-keyword set was surfaced by the corpus harness in [#54](https://github.com/axonops/mask/issues/54) and fixed in [#70](https://github.com/axonops/mask/issues/70); fourteen new table-driven cases in `rules_technology_test.go::TestApply_ConnectionString_OAuthAndCloudSecrets` pin the redaction, with corpus fixtures in `tests/corpus/connection_string.txt` covering both well-formed authority URLs and mixed-with-non-secret-params shapes. Matching remains case-insensitive and percent-decodes the raw key. ([#70](https://github.com/axonops/mask/issues/70))
- Tightened workflow token permissions to a `contents: read` workflow-level baseline across `cla.yml`, `contributors.yml`, `dependabot-automerge.yml`, and `release.yml`. Jobs that need elevated scopes (`contents: write`, `pull-requests: write`, `statuses: write`) now declare them per-job rather than workflow-wide, so a future job added to one of these workflows inherits read-only by default. Dropped the redundant `actions: write` scope from the CLA Assistant flow. Governance test `TestGovernance_WorkflowsLeastPrivilegeBaseline` pins the baseline (single `contents: read` key) across all six workflow files. ([#66](https://github.com/axonops/mask/issues/66))
- Pinned every GitHub Actions reference in `.github/workflows/` by 40-hex commit SHA with a trailing `# vX.Y.Z` tag comment. Covers 39 `uses:` lines across `ci.yml`, `release.yml`, `cla.yml`, `contributors.yml`, `dependabot-automerge.yml`, and `scorecard.yml`. Closes a tag-retag supply-chain hole — most critically on `cla.yml`'s `contributor-assistant/github-action` ref, which runs in `pull_request_target` context with the `CLA_ASSISTANT_PAT` admin token in env. `.github/dependabot.yml` now declares a `github-actions-all` group so all action bumps land in a single weekly PR with SHA and trailing comment updated together. Governance test `TestGovernance_AllActionsSHAPinned` walks every `.github/workflows/*.yml` file and rejects any unpinned `uses:` line or missing version comment. ([#65](https://github.com/axonops/mask/issues/65))

## Upgrading

From `v1.0.0` onwards `mask` follows the standard Go semantic-versioning
compatibility promise: breaking changes to the public API only in a new
major version. Minor and patch releases are always backwards-compatible
for the API surface documented on [pkg.go.dev](https://pkg.go.dev/github.com/axonops/mask).
Pin a specific tag in your `go.mod`, review the release notes for the
target version, and run your test suite with `-race` against the new
version before rolling to production.

## [1.0.0] — 2026-04-19

Initial public release.

### Added

- **Core API.** `mask.Apply(name, value)` for rule lookup and invocation, `mask.Register(name, fn)` for custom rules on the package-level registry, `mask.New(opts...)` for per-instance isolated registries, and `mask.Rules()` / `mask.Describe(name)` for runtime catalogue discovery.
- **Configurable mask character.** `mask.SetMaskChar(c)` for the package-level default and `mask.WithMaskChar(c)` for per-instance override.
- **Utility primitives** — direct-call helpers (`FullRedact`, `Nullify`, `SameLengthMask`, `KeepFirstN`, `KeepLastN`, `KeepFirstLast`, `PreserveDelimiters`, `ReducePrecision`, `DeterministicHash`) and factory wrappers (`KeepFirstNFunc`, `KeepLastNFunc`, `KeepFirstLastFunc`, `PreserveDelimitersFunc`, `ReplaceRegexFunc`, `ReducePrecisionFunc`, `FixedReplacementFunc`, `DeterministicHashFunc`). Regex masking is factory-only: a compiled `RuleFunc` is the supported form; a one-shot `ReplaceRegex(v, pattern, replacement)` helper that re-compiled on every call is intentionally not part of the public API.
- **Deterministic hashing** with configurable algorithm (`SHA256`, `SHA512`, `SHA3_256`, `SHA3_512`) and atomic keyed-salt configuration via `WithKeyedSalt(salt, version)` — the salt and version are validated together so keyed hashing can never be half-configured. Output format `<algo>:<hex16>` unsalted, `<algo>:<version>:<hex16>` when keyed. An empty salt or invalid version fails closed to `[REDACTED]` rather than silently downgrading.
- **68 built-in masking rules**: 4 utility primitives (`full_redact`, `same_length_mask`, `nullify`, `deterministic_hash`) plus 64 domain rules across identity (11 global + 14 country-specific), financial (11), healthcare (5), technology (14), and telecom + location (9). Every rule is fail-closed, honours the configured mask character, and is registered with a populated `RuleInfo` including category, jurisdiction, and an `input → output` example.
- **Runnable godoc examples** covering the simplest `Apply` case, fail-closed behaviour, custom rule registration, per-instance mask-character override, global mask-character override, direct primitive calls, factory-based registration, runtime discovery via `Describe`, and a realistic structured-log redaction sample.
- **Package documentation** (`doc.go`) covering the quick start, design principles, thread-safety contract, mask-character configuration, and explicit non-goals.
- **BDD coverage** under godog strict mode (`Strict: true`) with feature files per category (`identity.feature`, `financial.feature`, `health.feature`, `technology.feature`, `telecom.feature`, `country.feature`, plus `core_api.feature` and `primitives.feature`).
- **Benchmarks** for every rule, with allocation reporting and fail-closed-path variants where meaningful.
- **Drift guards.** CI-enforced checks that: every registered rule appears in exactly one README rule-table row; every rule carries an `Example:` line in its description; the BDD suite runs in strict mode; all Makefile targets are documented; no local paths or replace directives leak into `go.mod`; no AI-tooling attribution appears in commits, PRs, or code.
- **CI/CD.** Format check, vet, lint, unit and BDD tests, coverage, module tidy, security scan, cross-platform builds (`linux/amd64`, `darwin/arm64`, `windows/amd64`), and a CI-only release workflow — no local tagging permitted.
- `CONTRIBUTING.md`, `SECURITY.md`, `LICENSE` (Apache 2.0).
